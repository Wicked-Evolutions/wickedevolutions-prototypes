The three plugins

Each layer has one job.

Three focused plugins. One coordinated system. Remove any one and the others still exist — add all three and WordPress becomes a structured interface for AI.

Bridge · Node.js
abilities-mcp
v1.2.0 · @wicked-evolutions/abilities-mcp

The STDIO bridge that runs on your machine. Connects your AI client to WordPress over SSH or HTTP. Handles multi-site routing, session management, and three local bridge tools for health, browsing, and loading. Install once, connect to any number of sites.

3
bridge tools
Transport · PHP
abilities-mcp-adapter
v1.0.8 · WordPress plugin

The WordPress-side MCP protocol handler. Translates every registered ability into an MCP tool. Exposes a REST endpoint, handles the discovery gate, executes abilities with full permission checking, and supports batch execution. Upload and activate — your site becomes an MCP server.

2
public abilities
Engine · PHP
abilities-for-ai
v1.2.0 · WordPress plugin

The core engine. Registers 137 abilities across 21 WordPress modules — content, blocks, meta, menus, themes, users, filesystem, cron, revisions, site health, and more. Ships the Knowledge Layer: persistent AI memory in your WordPress database across sessions.

The Knowledge Layer

Persistent AI memory, inside WordPress.

Not an external service. Not a config file. Database tables — kl_* — that the AI reads and writes every session. The next session picks up where this one ended.

Behavioral directive — every connection

allowed: [ "read" ]
restricted: [ "write", "delete" ]
