01 — The three plugins

Each layer has one job.

Three focused plugins. One coordinated system. Remove any one and the others still exist — add all three and WordPress becomes a structured interface for AI.

Bridge · Node.js
abilities-mcp
v1.2.0 · @wicked-evolutions/abilities-mcp · npm

The STDIO bridge. Runs on your machine, connects your AI client to WordPress over SSH or HTTP. Handles multi-site routing and three local bridge tools for health checking, browsing, and selective loading. Install once, point at any number of sites.

3
bridge tools
Protocol · PHP
abilities-mcp-adapter
v1.0.8 · WordPress plugin · PHP 8.0+

The WordPress-side MCP handler. Translates every registered ability into an MCP tool, exposes a REST endpoint, runs the discovery gate, and executes abilities with full permission checking. Upload, activate — your site is an MCP server.

2
public abilities
Engine · PHP
abilities-for-ai
v1.2.0 · WordPress plugin · PHP 8.0+

The core engine. Registers 137 abilities across 21 WordPress modules — content, blocks, meta, menus, themes, users, filesystem, cron, revisions, site health and more. Ships the Knowledge Layer: persistent AI memory in your WordPress database, across sessions.

05 — The Knowledge Layer

Persistent AI memory, inside WordPress.

Not an external service. Database tables — kl_* — inside your WordPress. The AI reads and writes every session. The next session picks up where this one ended.

Behavioral directive — every connection
mode: inquiry
allowed: [ "read" ]
restricted: [ "write", "delete" ]
lifts: when the user gives a directive

The AI arrives knowing the site's full history. It reads everything. It writes nothing until you say go.
