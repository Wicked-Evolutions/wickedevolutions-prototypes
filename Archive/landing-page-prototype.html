How It Works

The sovereign stack.

Built on official WordPress core APIs. Not a hack — WordPress as intended.

Layer 1 — Your AI
Any MCP Client

Layer 2 — Transport
SSH Bridge or HTTP Bridge
SSH tunnel (key auth, zero credentials) or Streamable HTTP REST (Application Passwords). Your choice.

Official WordPress core feature. wp_register_ability() — the same API the WordPress team built.
↓
Layer 4 — Discovery

49 lines of PHP. Auto-discovers all registered abilities and serves them as MCP tools.
↓
Layer 5 — Abilities
291 Registered Abilities
WordPress Core · Fluent Suite · Astra · Spectra — composable plugins, each MIT/GPL licensed.
↓
Layer 6 — Your Infrastructure
Your Server. Your Data. Your Keys.
Any hosting. $10–30/month. You own every byte. No vendor can revoke your access.
Architecture

Native first. Headless if you must.
Sovereign always.

The abilities work at every tier. The question is how much frontend complexity you actually need.

90%
Native WordPress + AI

Gutenberg IS a React SPA (since 2018). Interactivity API adds frontend reactivity. Native blocks + CSS custom properties + AI = SPA-quality UX. No page builder. No npm. No build pipeline. One server.

For: Solopreneurs, creators, agencies, most WordPress sites
8%
Headless + AI Backend

WordPress as sovereign JSON API. React/Next.js/Astro frontend. AI operates WordPress through the same 291 abilities. Your data stays on your server even when the display layer is separate.

For: Custom interactive apps, enterprise with React teams
2%
Micro-frontends

Multiple specialized UIs consuming one WordPress abilities backend. Mobile app + dashboard + embedded widgets — same sovereign operational layer, different interfaces.

For: Multi-platform products with shared backend

Two bridges. One protocol.
Your choice.

SSH for maximum security. HTTP for maximum flexibility. Both production-tested.

SSH Bridge v2.3.0

For local development, agencies, and maximum security. Private key authentication — zero credentials stored anywhere.

✓ SSH key auth — private key never leaves your machine
✓ No REST endpoint exposed to the internet
✓ Auto-reconnect with request queuing
✓ Exponential backoff + health pings
✓ --register auto-configures Claude Desktop
✓ One SSH key → unlimited sites

HTTP Bridge v1.0.0

For remote teams, CI/CD pipelines, and cloud-hosted WordPress. Streamable HTTP REST with WordPress Application Passwords.

✓ Streamable HTTP transport (MCP spec 2025-11-25)
✓ WordPress Application Passwords — native auth
✓ Automatic session management
✓ Works without SSH access to server
✓ Zero dependencies — single file
✓ HTTPS encrypted, permission-scoped
