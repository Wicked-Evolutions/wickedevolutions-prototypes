Key execution mechanics
🔍
No wp_execute_ability()
Execution goes through the ability object — $ability→execute($input) — not a global function. The Adapter retrieves the object then calls execute directly.
wp_get_ability($name)→execute($input)
✓
Schema validation is automatic
WordPress Core validates $input against the registered input_schema before the callback is invoked. Your callback never receives invalid data.
input_schema · WP_Error on failure
⛔
Permission gate fires inside callback
The Registrar wraps every callback with abilities_for_ai_ability_enabled(). The ability always registers — it returns WP_Error 403 at execution time if disabled.
WP_Error · code: ability_disabled · 403
🪝
Registration hook is critical
Abilities must register on wp_abilities_api_init. Registering on any other hook silently fails — the registry isn't initialised yet. No error, no log.
add_action('wp_abilities_api_init', ...)
