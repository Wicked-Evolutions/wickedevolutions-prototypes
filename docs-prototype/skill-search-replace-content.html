Search & Replace Content

Bulk find-and-replace text across WordPress post content. Always preview with dry_run first.

practical content search-replace bulk

When to Use

  • URL changes after post type conversion or permalink structure change
  • Brand name changes across existing content
  • Fixing repeated typos or outdated references
  • Migrating domain references between environments

What It Does

Scans post_content across posts (up to 500 per call). Plain text matching, case-sensitive, exact match — not regex, by design. Dry run mode previews replacements without saving. Filter by post_type or specific post_ids. Returns per-post detail: ID, title, type, and replacement count.

Procedure

1 Always dry run first 
content-search-replace(
  search: "old-domain.com",
  replace: "new-domain.com",
  dry_run: true,
  site: "wicked.community"
)

Review the response: posts_scanned (how many checked), posts_affected (how many matched), total_replacements (match count), and details (per-post breakdown).

2 Scope if needed

Limit to a specific post type:

content-search-replace(
  search: "old text",
  replace: "new text",
  dry_run: true,
  post_type: "post"
)

Or limit to specific posts:

content-search-replace(
  search: "old text",
  replace: "new text",
  dry_run: true,
  post_ids: [9, 22, 25]
)

post_ids overrides post_type — if you provide both, only post_ids is used.

3 Execute

Once the dry run looks correct, run without it:

content-search-replace(
  search: "old text",
  replace: "new text"
)
4 Verify

Pick a post from the results and spot-check:

content-get(id: 19)
5 Flush cache

After bulk content changes, clear the page cache:

cache-flush-page-cache()

Gotchas

1
Case-sensitive. "Wicked" won't match "wicked". Run separate passes for different cases if needed.
2
Exact match, not regex. This is intentional — regex in a bulk content tool is dangerous for AI agents. If you need regex, use SSH (dev agent only).
3
Identical strings rejected. The search and replace values must differ.
4
Matches inside block markup. The search operates on raw post_content, which includes block comments (<!-- wp:... -->). A search for "center" will match alignment attributes, not just visible text. Be specific with your search strings.
5
500 post limit per call. For sites with more content, filter by post_type or run multiple passes.
6
No undo. To revert, run the inverse replacement. Keep the original search/replace strings until you've verified.

Tested On

community.wickedevolutions.com 2026-02-28
  • Dry run: found "Wicked Evolutions" in 7 of 9 posts (9 replacements)
  • Real replace + revert: roundtrip on post ID 19, 2 replacements each direction
  • Post type filter: limiting to "post" correctly excluded pages
  • Edge cases: identical strings rejected, post_ids filter works
